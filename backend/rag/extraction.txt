1. Context
Modern organizations generate large volumes of unstructured information spread across
many document formats and storage systems: contracts, reports, procedures, invoices,
presentations, scanned files, internal wikis. Finding a specific piece of information inside
this mass is slow, manual, and error-prone.
Classic keyword search fails because it matches words, not meaning. Generic AI chatbots
fail because they don’t know the company’s internal documents and tend to invent answers.
The goal of this internship is to build an intelligent platform that centralizes enterprise
knowledge, indexes documents automatically, and answers questions in natural language —
grounded exclusively in the organization’s own documents, with source citations.
The technical foundation is Retrieval-Augmented Generation (RAG): retrieve the relevant
passages first, then let the LLM answer using only those passages.
2. Objective
Design, develop, and deploy a functional web platform that allows a company to:
1. Upload and organize its internal documents.
2. Automatically process, index, and vectorize them.
3. Search them semantically, in natural language, in several languages.

4. Ask questions to an AI assistant that answers with sources, never from imagination.
5. Administer users, collections, and models through a dashboard.
6. Measure usage and knowledge coverage through analytics.
The intern is expected to work as a junior R&D engineer would in a real software company:

3. Functional Scope
3.1 Document Management
Upload, organize, and version documents in the following formats:
PDF
Word (DOCX)
Excel (XLSX)
PowerPoint (PPTX)
Plain text / Markdown
Images (via OCR)
Web pages (URL ingestion)
Documents are grouped into knowledge collections (e.g. HR, Legal, Technical).
3.2 Data Processing Pipeline
An automated, asynchronous pipeline that:
Extracts raw content from each format (including OCR for scanned files)
Cleans and normalizes the text
Splits documents into semantic chunks (with overlap strategy)
Generates embeddings for each chunk
Stores vectors + metadata (source file, page, section, collection, permissions)
3.3 AI Search Engine
Semantic (vector) search

10. Industrial Perspective
This is not a throwaway academic exercise. Once functional, the platform can be offered as
a SaaS solution to organizations with heavy internal documentation:
law firms, accounting firms, hospitals, universities, hotels, insurance companies,
manufacturers, public administrations, and SMEs.
Future extensions — Microsoft 365, Google Drive, SharePoint, Slack, Teams and email
integrations — would turn it into a complete enterprise knowledge assistant.
The intern is therefore contributing to a real product foundation, and is expected to treat it
as such.
Profile sought: engineering / master’s student in computer science, software engineering,
or data science. Comfortable with Python and web development. Curious about LLMs and
applied AI. Rigorous, autonomous, and able to document their work.
